## Manual QA Rubric

Score each item from 1 to 10. The pass threshold is shown in parentheses.

* Naturalness (8+): Does it sound written by a Mongolian product writer, not translated?
* Lived-moment recall (8+): Can a user remember a real moment from the wording?
* Option speakability (8+): Could a user say the option out loud?
* Engine invisibility (9+): Are internal tags and engine terms hidden from user-facing copy?
* Trust (8+): Does it avoid shame, pressure, and overclaiming?
* Safety calmness (9+): Is safety copy calm, clear, and non-diagnostic?
* Brevity (8+): Is the copy short enough for mobile but still meaningful?
* Report warmth (8+): Does the report start from life before naming a pattern?
* Commercial naturalness (8+): Does paid/lead copy sound concrete rather than operator-facing?
* Mobile readability (8+): Does it fit at 390px without dense paragraphs?

## Sprint 20A — Public Language Purge

Sprint 20A removed obvious English, mixed-language, and product-engine wording from public visible UI while keeping internal mechanism keys, scoring, tags, safety routing, pricing, fake payment behavior, and mock backend logic unchanged.

Removed or hidden public terms:

* `pattern`, `trigger`, `diary`, `reward`, `default`, `leverage point`
* `Preview`, `sample report`, `Demo unlock хийх`
* `Executive Load Failure`, `low-friction default`, `Decision fatigue`, `delivery/snack`
* `Food-as-Regulation`, `Decision-Default`, `Hunger-Safety`
* `fake payment`, `validation`, `purchase intent`, `lead capture`

Context-sensitive Mongolian replacements:

* `pattern` -> `давтамж`, `хэв маяг`, `зураглал`
* `trigger` -> `эхэлдэг нөхцөл`, `идэх хүсэл эхэлдэг нөхцөл`
* `diary` -> `тэмдэглэл`
* `reward` -> `таатай мэдрэмж авах хэрэгцээ`, `өөрийгөө баярлуулах хүсэл`
* `default` -> `бэлэн сонголт`, `хамгийн бага хүч шаарддаг сонголт`
* `leverage point` -> `эхэлж өөрчлөх хамгийн амар цэг`
* `Executive Load Failure` -> `Оройн шийдвэрийн ачаалал`
* `Decision fatigue` -> `олон шийдвэрийн дараах ядаргаа`
* `delivery/snack` -> `хоол захиалга / зууш`
* `sample report` -> `тайлангийн жишээ`

Public vs internal distinction:

* Internal mechanism names, regex strings, tags, score keys, and metadata remain in code for scoring and tests.
* Public reports use `publicMechanismName`, `publicMechanismShort`, and `publicMechanismHiddenFunction` copy instead of exposing raw mechanism labels.
* The one-time paywall preview no longer exposes internal mechanism names such as `Executive Load Failure`.
* Validation summary product labels display as Mongolian labels and prices display with `₮`.

Demo mode visibility rule:

* Public/default UI hides demo unlock controls.
* Demo unlock functionality remains available through explicit internal/demo mode (`demoMode` in tests or `?demo=1` / `?internal=1` in browser).
* `tests/public-language-purge.test.js` verifies public screens hide demo/validation/fake-payment language and confirms demo unlock still works in demo mode.
